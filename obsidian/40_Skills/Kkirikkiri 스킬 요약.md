# Kkirikkiri 스킬 요약

## 용도
- 자연어 한마디로 인터뷰 → 팀 구성 → 실행 → 검증까지 이어지는 자동 팀 빌더 스킬.

## 핵심 포인트
- 단순 답변용이 아니라 실제 팀을 짜고 움직이는 워크플로우다.
- 인터뷰, 환경 스캔, 역할 분해, 공유 메모리, 검증 루프까지 포함한다.
- 결과 품질이 부족하면 2라운드 재구성까지 고려한다.

## 자주 쓰는 흐름
1. 사용자 목표 파악
2. 프리셋 매칭 + 인터뷰
3. 환경 스캔
4. 팀 제안 후 승인
5. 팀 생성 + 공유 메모리 초기화
6. 실행 + 품질 검증

## 주의사항
- 확인 없이 팀 생성하면 안 된다.
- 공유 메모리 없이 팀을 돌리면 컨텍스트가 쉽게 무너진다.
- 팀장은 직접 구현보다 분배/검증/통합에 집중해야 한다.
