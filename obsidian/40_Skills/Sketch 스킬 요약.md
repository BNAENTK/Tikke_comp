# Sketch 스킬 요약

## 용도
- 실제 구현 전에 2~3개의 UI 방향을 disposable HTML mockup으로 빠르게 비교할 때 쓰는 스킬.

## 핵심 포인트
- 목적은 정답 하나를 만드는 게 아니라, 서로 다른 design stance를 비교하는 것이다.
- 각 variant는 standalone HTML 파일이어야 한다.
- 작성 후 browser로 실제 렌더를 보고 고쳐야 한다.

## 자주 쓰는 흐름
1. 화면의 vibe / reference / core action 파악
2. 2~3개 variant 생성
3. 각 variant를 브라우저로 시각 확인
4. 비교표와 추천안 정리

## 주의사항
- 색만 조금 다른 2개 버전은 의미가 없다.
- static screenshot 수준이면 sketch 가치가 떨어진다.
- production code처럼 과하게 다듬는 것도 낭비다.
