# Systematic Debugging 스킬 요약

## 용도
- 버그, 테스트 실패, 이상 동작을 고칠 때 근본 원인부터 찾게 강제하는 디버깅 스킬.

## 핵심 포인트
- 핵심 원칙은 `원인 파악 전 수정 금지`다.
- 재현, 최근 변경 확인, 증거 수집, 데이터 흐름 추적, 가설 검증, 마지막 수정 순으로 간다.
- 3번 이상 수정 시도가 실패하면 아키텍처 문제를 의심해야 한다.

## 자주 쓰는 흐름
- 에러 메시지 정독
- 재현 명령 확보
- git diff / 최근 변경 확인
- 호출 흐름 추적
- 최소 가설 테스트
- 회귀 테스트 추가 후 수정

## 주의사항
- 급하다고 바로 패치하면 오히려 오래 걸린다.
- 증상만 없애는 fix는 재발 가능성이 높다.
- 여러 가설을 한 번에 섞어 고치면 원인 판단이 어려워진다.
