# Insane Search 스킬 요약

## 용도
- 일반 웹 검색으로 막히는 사이트, WAF/차단이 있는 사이트, X/Reddit/YouTube/네이버 같은 플랫폼 접근을 우회할 때 쓰는 스킬.

## 핵심 포인트
- 단순 web fetch가 막히면 즉흥적으로 헤더를 꾸미기보다 정해진 fetch chain을 타는 것이 핵심이다.
- 공식 공개 API가 있는 플랫폼은 먼저 Phase 0 경로를 본다.
- WAF 사이트는 성공 판정을 엄격하게 해야 한다. HTTP 200만으로 성공 처리하지 않는다.

## 자주 쓰는 흐름
- URL/플랫폼 유형 분류
- 공식 API 또는 JSON 엔드포인트 우선 확인
- generic fetch chain 실행
- 필요 시 Playwright fallback
- trace 기반으로 재시도 판단

## 주의사항
- 사이트별 하드코딩보다 일반화된 패턴을 우선한다.
- 첫 200 응답에서 성급히 성공 선언하면 안 된다.
- 막힌 URL 접근 문제에서만 쓰고, 평범한 검색에는 과하게 쓰지 않는다.
