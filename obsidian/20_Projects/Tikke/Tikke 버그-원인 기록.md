# Tikke 버그-원인 기록

`tikke`에서 반복되는 문제와 근본 원인을 짧게 축적하는 노트.

## 기록 원칙

- 증상보다 **원인** 중심으로 적는다.
- 재현 조건, 영향 범위, 확인 방법을 짧게 남긴다.
- 코드 원문, 긴 로그, secret은 넣지 않는다.

## 현재 알려진 패턴

### 오버레이 미리보기/이벤트 반영 안 됨
- 증상: 오버레이가 열려도 미리보기나 gift/message 반영이 되지 않음.
- 근본 원인: `top-donor`류 HTML에 `window message` 핸들러가 빠져 있으면 미리보기/이벤트 반영이 깨질 수 있다.
- 영향 범위: 오버레이 미리보기, 방송 중 이벤트 반영 확인 흐름.
- 확인 방법: 오버레이 HTML에 메시지 핸들러 존재 여부와 preview 동작을 같이 확인.
- 관련 노트: [[Tikke 오버레이 규칙]]

### 새 오버레이가 한쪽 환경에서만 보임
- 증상: 새 오버레이를 추가했는데 desktop 또는 worker 공개 경로 한쪽에서만 보임.
- 근본 원인: 새 오버레이 HTML을 desktop + worker/public/overlay 양쪽에 추가하지 않으면 경로가 어긋난다.
- 영향 범위: 로컬 미리보기와 배포 환경 간 불일치.
- 확인 방법: 두 위치에 파일이 모두 들어갔는지, 접근 경로가 맞는지 확인.
- 관련 노트: [[Tikke 오버레이 규칙]]

### 오버레이 UI는 보이는데 실제 진입 링크가 비어 있음
- 증상: UI에서 오버레이 카드/설정은 보이지만 실제 진입이나 미리보기 링크가 동작하지 않음.
- 근본 원인: `cloudUrl`, `previewUrl`, 오버레이 HTML 연결 중 하나가 누락되면 끝까지 연결되지 않는다.
- 영향 범위: 설정 화면, 미리보기 링크, 실제 오버레이 접근.
- 확인 방법: UI 추가 후 `cloudUrl`, `previewUrl`, HTML 경로를 한 세트로 점검.
- 관련 노트: [[Tikke 오버레이 규칙]]
