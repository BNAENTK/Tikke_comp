# Tikke Claude Code 프롬프트 템플릿

`tikke` 작업에서 `Claude Code`에 바로 복붙해 쓰기 위한 프롬프트 템플릿 모음.

## 사용 원칙

- 목표, 수정 대상, 금지사항, 검증 기준을 같이 넣는다.
- 오버레이 작업이면 `preview/link/event` 검증을 꼭 포함한다.
- 버그 수정이면 증상만 말하지 말고 재현 조건을 같이 넣는다.
- 결과 보고 형식도 같이 고정하면 재사용성이 올라간다.

## 1) 기능 추가 템플릿

```text
작업 목표:
- [여기에 추가할 기능 한 줄]

배경:
- 프로젝트: tikke
- 관련 맥락: [필요한 짧은 설명]

수정 대상 후보:
- [파일/폴더 경로]
- [파일/폴더 경로]

요구사항:
- [필수 요구사항 1]
- [필수 요구사항 2]
- [필수 요구사항 3]

금지사항:
- 버전을 내리지 말 것
- 불필요한 광범위 리팩터링 금지
- Obsidian용 문서에 소스코드 원문 복붙 금지

검증:
- 관련 화면/기능이 실제로 동작하는지 확인
- 오버레이 작업이면 preview/link/event 반영까지 확인
- 영향받는 기존 경로에 회귀 없는지 확인

결과 보고 형식:
- 수정한 파일
- 핵심 변경점
- 검증 결과
- 남은 리스크
```

## 2) 버그 수정 템플릿

```text
버그 목표:
- [무슨 버그를 고칠지 한 줄]

현재 증상:
- [보이는 증상]

기대 동작:
- [원래 어떻게 동작해야 하는지]

재현 조건:
- [언제/어떻게 재현되는지]

수정 대상 후보:
- [파일/폴더 경로]

요청:
- 먼저 원인 후보를 좁히고
- 최소 수정으로 해결하고
- 비슷한 경로에 회귀가 없는지 같이 확인해줘

금지사항:
- 원인 파악 전 감으로 대규모 수정하지 말 것
- 로그만 늘리고 끝내지 말 것

검증:
- 같은 재현 조건에서 증상이 사라졌는지 확인
- 관련 경로가 깨지지 않았는지 확인

결과 보고 형식:
- 원인
- 수정 내용
- 검증 결과
- 추가로 의심되는 부분
```

## 3) 오버레이 작업 템플릿

```text
작업 목표:
- [오버레이 추가/수정 목표]

수정 대상 후보:
- [desktop 쪽 경로]
- [worker/public/overlay 쪽 경로]
- [UI 연결 경로]

반드시 확인할 것:
- desktop + worker/public/overlay 양쪽 반영
- `cloudUrl`, `previewUrl`, HTML 연결 확인
- `top-donor`류면 `window message` 핸들러 확인
- preview 화면 실제 동작 확인
- 이벤트/메시지 반영 확인

금지사항:
- 한쪽 경로만 수정하고 끝내지 말 것
- preview 확인 없이 완료 처리하지 말 것

결과 보고 형식:
- 수정 파일
- 연결한 경로
- preview 확인 여부
- 이벤트 반영 확인 여부
- 남은 리스크
```

## 4) 리팩터링 템플릿

```text
작업 목표:
- [정리/리팩터링 목표]

범위:
- [허용 범위]
- [건드리지 말아야 할 범위]

의도:
- 동작은 유지하면서 [가독성/중복 제거/구조 개선] 중심으로 정리

금지사항:
- 기능 동작 변경 금지
- 범위 밖 수정 금지
- 검증 없는 대규모 이동 금지

검증:
- 기존 동작 유지 확인
- 영향 경로 확인

결과 보고 형식:
- 정리한 포인트
- 유지된 동작
- 혹시 남은 기술부채
```

## 5) PR 정리 템플릿

```text
목표:
- 현재 변경사항을 PR 가능한 상태로 정리

할 일:
- 변경사항 요약
- 불필요한 변경 제거 여부 점검
- 리뷰 포인트 정리
- 테스트/검증 결과 정리

결과 보고 형식:
- PR 제목 후보
- PR 본문 초안
- 검증 결과
- 리뷰가 필요한 포인트
```
