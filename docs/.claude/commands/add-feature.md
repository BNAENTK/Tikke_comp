기존 Tikke 프로젝트에 새 기능을 추가한다.

절차:
1. 요청한 기능이 `apps/desktop`, `apps/web`, `apps/worker`, `packages/shared`, `services/qwen-tts` 중 어디에 속하는지 분류한다.
2. 관련 기존 파일을 찾고 현재 구현 패턴을 파악한다.
3. 최소 변경 계획을 세운다.
4. 필요한 파일만 수정한다.
5. 변경 범위에 맞는 타입 체크/빌드/하네스 검증을 실행한다.
6. 변경 파일, 변경 이유, 검증 결과를 한국어로 요약한다.

주의:
- 새 프로젝트를 만들지 않는다.
- 기존 구조와 스타일을 유지한다.
- 사용자가 요청하지 않은 대규모 리팩토링은 하지 않는다.
- secret 값을 출력하지 않는다.
